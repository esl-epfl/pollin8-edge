3. Focal loss: stop wasting effort on easy background

Focal loss down-weights easy examples. In your task, many tiles are easy background, while the important cases are tiny insects and mimic species.

Move γ to see the effect

Easy example
Hard example

Plain-English meaning

With ordinary BCE, millions of easy “not insect” regions still contribute learning signal. Focal loss says: if the model already gets this example right, reduce its importance.

focal multiplier ≈ (1 − p)γ

This toy display uses the common focal intuition: higher γ makes easy examples almost disappear from the loss. Hard cases keep more influence.

Why it matters: better recall on rare/tiny insects means fewer under-counts.

4. Class-balanced weighting: rare species should matter more

If one species has 1,000 examples and another has 42, the model naturally learns the common one better. Class-balanced loss gives rare classes more voice without exploding their weights.

Edit the class counts

effective number = (1 − βⁿ) / (1 − β)

Normalized class weights

Weights are normalized so their average is 1. Rare classes get boosted, but not as violently as raw inverse frequency.

5. NWD vs IoU: why tiny boxes are fragile

IoU measures box overlap. For tiny insects, a one-pixel shift can collapse overlap. NWD is designed to be more scale-robust by comparing boxes more like soft Gaussian blobs.

6. Training metric: choose the best model for the real job

YOLOv5 often chooses checkpoints using a fitness score that heavily rewards mAP@0.5:0.95. That metric rewards tight boxes. Your actual task needs accurate centre-matched detections and counts.

Columns = predictions

Diagonal = correctColumn errors = false positivesRow errors = false negatives
For your insect task: a row with many off-diagonal cells means that true species is being missed or confused. A column with many off-diagonal cells means the model over-predicts that species.

Per-class metrics

TP is the diagonal cell for the class. FN is the rest of that class’s row. FP is the rest of that class’s column.

precision = TP / (TP + FP)
recall = TP / (TP + FN)
F₁ = 2·precision·recall / (precision + recall)

Macro vs weighted vs micro

Macro-F₁ treats every species equally, so rare species matter a lot.

Weighted-F₁ gives common species more influence.

Micro-F₁ pools all decisions. In ordinary single-label multiclass classification, it equals accuracy.

Important for rare species: accuracy can look good while rare-species recall is terrible. That is why macro-F₁ and per-species count bias are important in your reporting.

Classification matrix

Assumes every object has one true class and one predicted class. Good for analyzing species confusion after matching detections to labels.

Detection matrix

Object detection often needs an extra missed column and false detection row.

Example: a true insect with no matched prediction contributes to “missed.” A prediction that matches no real insect contributes to “false detection.”

Final mental model

Focal

Focus learning on hard insects and mimics instead of easy empty background.

Class-balanced

NWD

Make tiny-box localization less brittle than pure IoU.

F₁ selection

Choose the checkpoint that best matches counting, not strict box aesthetics.
